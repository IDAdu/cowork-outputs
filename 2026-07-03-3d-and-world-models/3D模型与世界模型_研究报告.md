# 3D 生成、3D 表示与重建,及其与世界模型(World Models)的关系

**一份面向 AI 研究者的深度研究报告**

*编制日期:2026 年 7 月 · 覆盖文献 2018–2026*

---

## 摘要(TL;DR)

过去三年,三条原本相对独立的研究线正在快速合流:

1. **3D 表示与重建**从"逐场景隐式优化"(NeRF, 2020)走向"显式实时表示"(3D Gaussian Splatting, 2023),再走向"前馈、可泛化、免位姿"的统一几何网络(LRM / VGGT, 2024–2025)。
2. **3D 生成**从"用 2D 扩散先验做 SDS 优化"(DreamFusion, 2022)经"多视扩散缓解 Janus 问题"(MVDream 等),演进到"大规模原生 3D 扩散/流模型"(CLAY / TRELLIS / TripoSG / Hunyuan3D,2024–2025),秒级出资产已成常态。
3. **世界模型(World Models)**从 model-based RL 的潜空间想象(Ha & Schmidhuber 2018 → Dreamer 系)分化为两条路线:**生成式像素预测**(Sora / Genie / GameNGen)与**表征式潜空间预测**(JEPA)。同时,自动驾驶与具身智能领域出现了以 **3D occupancy / 点云 / 4D 高斯**为场景表示的世界模型。

**核心论点**:纯像素级的视频"世界模拟器"在受控与真实基准上均被证明**未真正学到物理规律**(PhyWorld, Physics-IQ),其泛化是"case-based"而非规则性的。这使得**显式 3D 几何一致性、可检视的结构化状态、以及 sim-to-real**成为世界模型走向具身智能的关键瓶颈——也正是 3D 表示/生成研究能够贡献的地方。Fei-Fei Li 的 renderer / simulator / planner 三分法为这一融合提供了清晰的概念框架:三者本质上共享同一套底层知识——**几何、物理、动力学**。

> **阅读提示**:本报告对"厂商公告/演示宣称"与"经同行评审或可复现验证的结果"做了明确区分。凡属前者均标注"[厂商声称]",在引用其量化指标(参数量、加速倍数、拒绝率等)时应保持谨慎。

## 第一部分:3D 表示与重建

### 1.1 Neural Radiance Fields(NeRF):奠基范式

**NeRF**(Mildenhall et al., ECCV 2020)用一个 MLP 表示连续的 5D 函数 `F(x, d) → (c, σ)`:输入 3D 坐标 `x=(x,y,z)` 与视角方向 `d=(θ,φ)`,输出发射颜色 `c` 与体密度 `σ`。沿相机光线做**经典体渲染(volume rendering)** 积分成像 `C(r)=∫ T(t)σ(t)c(t)dt`(`T(t)` 为累积透射率)。体渲染天然可微,因此仅凭一组已知位姿的图像即可优化。

两个关键设计:

- **位置编码(positional encoding)**:用 `γ(p)=(sin(2ᵏπp), cos(2ᵏπp))` 把低维坐标映射到高频空间,克服 MLP 的**谱偏差(spectral bias)**,使网络能拟合高频细节。
- **分层采样**:coarse + fine 两级网络,按密度重要性重采样。

**局限**:逐场景优化(不可泛化)、训练慢(单场景 V100 上约 1–2 天)、渲染慢(每像素上百次 MLP 查询,远非实时)。这三点定义了后续三年的优化方向。

### 1.2 NeRF 的加速与变体

| 方法 | 核心机制 | 关键主张 |
|---|---|---|
| **Instant-NGP** (SIGGRAPH 2022) | 多分辨率哈希编码(multiresolution hash encoding)+ 极小 MLP + fully-fused CUDA kernel | 数秒训练、数十毫秒渲染,较原始 NeRF 加速数个数量级(官方摘要确认) |
| **Mip-NeRF** (ICCV 2021) | 用圆锥台(conical frustum)+ 积分位置编码(IPE)做抗锯齿 | 比 NeRF 快约 7%、模型减半 |
| **Mip-NeRF 360** (CVPR 2022) | 非线性场景参数化 + proposal MLP + 失真正则,处理无界 360° 场景 | 常作为 3DGS 的对比基线(训练约 18–20 小时,二手来源) |
| **Plenoxels** (CVPR 2022) | **完全无神经网络**:稀疏体素网格 + 每体素球谐系数 | 质量与 NeRF 相当,训练加速约数百倍 |
| **DVGO** (CVPR 2022) | 稠密体素网格 + 浅层 MLP,直接优化 | 训练从数天降至约 15 分钟 |
| **TensoRF** (ECCV 2022) | 4D 辐射场张量的低秩分解(CP / VM) | CP 分解相对 Plenoxels 内存降约 200× |

**权衡小结**:Instant-NGP(哈希,最快、GPU 友好);Plenoxels/DVGO(显式网格,训练快但存储大);TensoRF(张量分解,内存-质量平衡最佳);Mip-NeRF 系列(质量/抗锯齿最优但慢)。

> *注:除 Instant-NGP 的"秒级训练/毫秒渲染"来自原文摘要外,上表若干加速倍数与训练时长来自二手综述,属中等置信度。*

### 1.3 3D Gaussian Splatting(3DGS):显式实时表示的转折点

**3D Gaussian Splatting**(Kerbl et al., ACM TOG / SIGGRAPH 2023)是近年 3D 视觉最重要的范式转变之一。三大要素:

1. **显式表示**:从 SfM 稀疏点云初始化,用一组各向异性 3D 高斯(位置、协方差、不透明度、球谐颜色)表示场景,保留连续体积辐射场特性,同时避免空区计算。
2. **交错优化 + 自适应密度控制**:动态克隆/分裂/剔除高斯。
3. **基于图块(tile-based)的可见性感知可微光栅化**:既加速训练又实现实时渲染。

**关键指标(原文摘要确认)**:在无界完整场景、**1080p 分辨率下 ≥100 FPS** 实时渲染,视觉质量达到 SOTA。

**与 NeRF 对比**:渲染约 100+ FPS(vs NeRF ~5 FPS 量级);训练"分钟级"(vs Mip-NeRF 360 数十小时);Mip-NeRF 360 数据集上 PSNR 与 NeRF SOTA 相当,各场景互有胜负。**代价**:显存/存储占用大(数百 MB 点云),失去隐式表示的紧凑性。

### 1.4 隐式 vs 显式表示:权衡全景

| 表示 | 内存 | 可编辑性 | 渲染速度 | 拓扑特性 | 代表工作 |
|---|---|---|---|---|---|
| 隐式 SDF / 占据 | 极省(MLP 权重) | 差(需重训) | 慢(查询密集) | 任意拓扑、水密 | DeepSDF, Occupancy Networks, NeuS |
| 神经辐射场(NeRF) | 省 | 差 | 慢 | 隐式软密度 | NeRF, Mip-NeRF |
| 点云 | 中 | 中 | 快(无连通性) | 无拓扑、易有孔洞 | Point-E |
| 网格(mesh) | 省-中 | 好(图形管线友好) | 极快(栅格化) | 拓扑固定、难变形 | — |
| 体素(voxel) | 大 `O(n³)` | 中 | 中 | 规则、分辨率受限 | — |
| 3D 高斯(显式) | 大 | 较好 | 极快(实时) | 无连通性、点基 | 3DGS |

三篇隐式表示奠基工作值得单列:**Occupancy Networks**(CVPR 2019,学习占据概率函数 `o: ℝ³→[0,1]`)、**DeepSDF**(CVPR 2019,连续回归符号距离函数,支持形状插值/补全)、**NeuS**(NeurIPS 2021,用无偏、遮挡感知的权重函数把 SDF 接入体渲染,几何精度优于 occupancy 路线且无需前景掩码)。

### 1.5 前沿(2024–2026):动态、大规模、前馈化

- **动态 / 4D Gaussians**:4D Gaussian Splatting(CVPR 2024)、Deformable 3D Gaussians(CVPR 2024)、Spacetime Gaussian(CVPR 2024)实现实时动态场景渲染。
- **大规模场景重建**:NeRF 路线的 Block-NeRF / Mega-NeRF / Switch-NeRF;3DGS 路线的 VastGaussian、CityGaussian(ECCV 2024)、CityGaussianV2(ICLR 2025)。
- **前馈 / 可泛化重建**(最重要的方向性转变):
  - **pixelSplat**(CVPR 2024)、**MVSplat**(ECCV 2024)从图像对前馈预测 3D 高斯;
  - **LRM / GS-LRM**(2024)单图/稀疏视图前馈出 3D;
  - **VGGT: Visual Geometry Grounded Transformer**(Wang et al., **CVPR 2025 Best Paper**, arXiv 2503.11651):单一前馈网络从 1 到数百视图**一秒内**直接推断相机参数、深度、点图、3D 点轨迹,**无需 bundle adjustment、特征匹配或迭代优化**。

**主线判断**:3D 重建正从 (1) 逐场景隐式优化(NeRF)→ (2) 显式实时表示(3DGS)→ (3) 前馈可泛化/免位姿(VGGT、LRM 系)演进;4D 动态与城市级大规模是两大扩展轴。这一"前馈化"趋势直接为世界模型提供了可实时、可泛化的 3D 场景表示能力。

## 第二部分:3D 生成

### 2.1 SDS 范式:用 2D 扩散先验优化 3D

核心思想:借助预训练的 2D text-to-image 扩散模型作为先验,通过 **Score Distillation Sampling(SDS)** 优化一个可微 3D 表示(NeRF/mesh),**无需 3D 训练数据**。

- **DreamFusion**(Poole et al., Google, ICLR 2023)——开山之作。SDS 将扩散模型的去噪梯度作为损失,优化 NeRF 使其各视角渲染匹配文本条件扩散先验(用 Imagen)。
- **Magic3D**(Lin et al., NVIDIA, CVPR 2023)——两阶段 coarse-to-fine:先用稀疏 3D hash grid 得粗模型,再用高分辨率 latent diffusion 优化带纹理 mesh(DMTet)。约 40 分钟出 mesh,较 DreamFusion 约 2× 加速。
- **ProlificDreamer**(Wang et al., THU, NeurIPS 2023)——提出 **Variational Score Distillation(VSD)**,把 3D 参数建模为随机变量(particle-based 变分框架),SDS 是其单粒子特例。在常规 CFG 下即可工作,解决 SDS 的**过饱和/过平滑/低多样性**,可得 512² 高保真 NeRF。
- **Fantasia3D**(Chen et al., SCUT, ICCV 2023)——**解耦几何与外观**:几何用 DMTet,外观引入空间变化 BRDF 做 PBR,可 relight、可接图形引擎。

**SDS 范式的三大顽疾**:① **Janus / 多面问题**(2D 先验缺乏 3D 一致性,物体出现多张脸);② **过饱和/过平滑**(需大 CFG);③ **优化极慢**(per-prompt 1–10 小时)。

### 2.2 多视扩散:从根源缓解一致性问题

思路是让扩散模型直接生成多视一致图像,再重建或作更强的 SDS 先验。

- **Zero-1-to-3 / Zero123**(Columbia, ICCV 2023):单图零样本新视角合成,把相对相机旋转作为条件,用 Objaverse 合成数据训练,对野生图像有强泛化。
- **MVDream**(ByteDance, ICLR 2024):在 UNet 中引入 inflated 3D self-attention 建模跨视依赖,其 multi-view SDS 大幅缓解 Janus。
- **SyncDreamer**(HKUST, ICLR 2024):synchronized multiview diffusion,用 3D-aware feature attention 在反扩散每步同步。
- **Wonder3D**(HKU, CVPR 2024):cross-domain diffusion 同时生成多视 normal + color,2–3 分钟出带纹理 mesh。

### 2.3 前馈 / 原生 3D 生成

- **Point-E**(OpenAI, 2022):文本→彩色点云,单 GPU 分钟级。
- **Shap-E**(OpenAI, 2023):生成隐式函数参数(可渲染为 mesh 或 NeRF)。
- **LRM(Large Reconstruction Model)**(Adobe/ANU, ICLR 2024):约 5 亿参数 transformer,DINO encoder + cross-attention 投影到 triplane NeRF,约 100 万对象训练,**5 秒**出 3D。
- **TripoSR**(Stability AI + Tripo, 2024):基于 LRM 的开源前馈重建,单图 **<0.5 秒**出 mesh(MIT 许可)。
- **原生 3D latent diffusion 谱系**(基于 3DShape2VecSet VAE latent):Michelangelo、Direct3D(NeurIPS 2024,无需多视扩散或 SDS 的原生 3D 生成)、CraftsMan3D、TripoSG、Step1X-3D 等。

### 2.4 基于 3D Gaussian Splatting 的生成

- **DreamGaussian**(ICLR 2024 Oral):首个把 3DGS 用于生成,progressive densification 收敛远快于 NeRF 的 occupancy pruning,较此前优化式方法**约 10× 加速**。
- **GaussianDreamer**(HUST, CVPR 2024):桥接 2D 与 3D 扩散,单 GPU 约 15 分钟出可实时渲染 3D。
- **LGM**(ECCV 2024 Oral):前馈生成多视高斯,出高分辨率 3D。
- **GRM**(ECCV 2024):前馈把 sparse-view 像素翻译为 pixel-aligned Gaussians,约 **0.1 秒**重建。

### 2.5 前沿(2024–2026):大规模原生模型与 mesh 生成

**大规模原生 3D 生成模型**(均有论文支撑,置信度高):

- **CLAY**(ShanghaiTech + Deemos, SIGGRAPH 2024):**1.5B 参数**几何生成器,multi-resolution VAE + latent DiT,支持 text/image 及 voxel/bbox/point cloud 等 3D-aware 控制,PBR 纹理 2K。
- **TRELLIS**(Microsoft, CVPR 2025 Spotlight):**Structured LATent(SLAT)** 统一表示,可解码为 Radiance Fields / 3D Gaussians / mesh 多种格式,最大约 2B 参数。
- **Hunyuan3D 2.0**(Tencent, 2025):首个全开源高质量 3D-DiT,含形状生成(flow-based DiT)+ 纹理合成两组件。
- **TripoSG**(VAST-AI/Tripo, 2025):large-scale rectified flow transformer + 先进 VAE,训练于 200 万 Image-SDF 对。

**Mesh 生成**(直接产出可用拓扑):**MeshGPT**(自回归生成 artist-created mesh)、**MeshAnything**(ICLR 2025,shape-conditioned 自回归 mesh,面数比 dense 提取少数百倍)。

**商业系统** [厂商声称,慎引]:Tripo AI、Meshy AI、Rodin / Hyper3D(Deemos)。这些系统的参数量与性能数字来自厂商公告与第三方评测,不宜作为硬指标。

## 第三部分:世界模型(World Models)

### 3.1 起源:Ha & Schmidhuber(2018)与 model-based RL

**"World Models"**(Ha & Schmidhuber, arXiv 1803.10122; NeurIPS 2018 版题为 *Recurrent World Models Facilitate Policy Evolution*)提出以无监督方式学习环境的**压缩时空表征**,再让极小的 controller 在潜表征上决策。三组件:

- **V(Vision)**:VAE,把高维观测压缩为潜向量 `z`;
- **M(Memory)**:MDN-RNN,学习潜状态时间演化;
- **C(Controller)**:以 `(z, h)` 为输入的极简线性策略,用 CMA-ES 训练。

标志性演示:agent 可完全在世界模型生成的**"幻想梦境(dream)"** 内训练策略,再迁移回真实环境。这一"在想象中学习"的思想上溯 Schmidhuber 1990s 的 controller/model 分离框架。

### 3.2 Dreamer 谱系:潜空间想象的主线

| 模型 | 发表 | 关键机制 / 里程碑 |
|---|---|---|
| **PlaNet** | ICML 2019 | 提出 **Recurrent State-Space Model(RSSM)**:潜状态拆为确定性分量 `h`(GRU)+ 随机分量 `z`;用 CEM/MPC 在线规划 |
| **Dreamer** | ICLR 2020 | 改为在 RSSM 潜空间**想象轨迹**中反传 value 梯度,联合训练 actor-critic |
| **DreamerV2** | ICLR 2021 | 随机潜变量改为 **discrete/categorical** 表示;首个仅在世界模型内学习即达 Atari human-level 的单 GPU agent |
| **DreamerV3** | **Nature 640, 647–653 (2025)**;arXiv 2301.04104 | 固定超参跨 150+ 任务、8 大领域超越专用方法;**首个无需人类数据或课程学习、从零在 Minecraft 采集钻石**的算法 |

DreamerV3 的通用性依赖一揽子稳健化技巧(symlog 预测、two-hot 编码、free bits、percentile return normalization),使同一套超参跨域可用。*(DreamerV3 已于 2025 年正式发表于 Nature,已核实。)*

### 3.3 JEPA:表征式世界模型——"在潜空间预测,而非像素"

Yann LeCun 在 2022 年立场论文 *A Path Towards Autonomous Machine Intelligence* 中提出 **JEPA(Joint-Embedding Predictive Architecture)**。核心论点:在**抽象表征空间**预测被遮蔽/未来部分,而非重建像素。理由:像素级重建被迫建模不可预测的细节(噪声、纹理),浪费容量;表征空间预测可丢弃不可预测细节,学到更语义、更利于规划的特征。

- **I-JEPA**(Meta, CVPR 2023):图像域,从 context block 在表征空间预测多个 target blocks,无需手工数据增强。
- **V-JEPA**(Meta, 2024):视频域,在潜空间预测被遮蔽的时空区域。
- **V-JEPA 2**(Meta / FAIR + Mila, **arXiv 2506.09985**, 2025-06):action-free JEPA,在**超过 100 万小时**视频上预训练;其 action-conditioned 变体 **V-JEPA 2-AC** 用 **<62 小时**无标注机器人视频(Droid)微调,在 MPC 规划循环内实现机器人操作,**zero-shot 部署到未见实验室**做 pick-and-place;motion understanding 在 SSv2 达 77.3% top-1。*(arXiv 号与指标已核实。)*

JEPA 代表了生成式视频世界模型的**主要架构对立面**——"世界模型 ≠ 视频生成"的学术阵营。

### 3.4 视频生成"作为世界模拟器":宣称与争议

**Sora**(OpenAI, 2024-02,技术报告 *Video generation models as world simulators*):text-conditional diffusion transformer,作用于视频/图像 latent 的 **spacetime patches**。OpenAI 论断:"扩展视频生成模型是通向通用物理世界模拟器的有希望路径",并称 3D、物体等属性"纯粹是规模的涌现现象"。**但"world simulator"是 OpenAI 的框定/宣称,非独立验证结论**;OpenAI 自陈局限:无法准确建模玻璃破碎等物理交互、长时序出现物体自发出现与不一致。Sora 2(2025-09)宣称物理更精确 [厂商声称,缺独立基准复核]。

**Genie 系列**(DeepMind):

- **Genie 1**(2024-02, ICML 2024 最佳论文, arXiv 2402.15391):**11B 参数** foundation world model,从 20 万+小时无标注游戏视频**无监督**训练。关键创新是 **latent action model(LAM)**——无需任何真值动作标签即实现逐帧可控。
- **Genie 2**(2024-12):从单张图生成可玩 3D 环境,一致性约 10–20 秒。
- **Genie 3**(2025-08)[能力指标为官方公告]:号称首个**实时交互**世界模型,24 fps / 720p,一致性维持数分钟,记忆约 1 分钟;2026-01 起通过 Project Genie 有限开放。

**神经游戏引擎**(有论文/开源支撑,置信度高):

- **GameNGen**(Google/DeepMind + 特拉维夫大学, 2024-08):"扩散模型即实时游戏引擎",实时模拟 DOOM,TPU 上 20 fps,next-frame PSNR 29.4;人类 5 分钟后区分真伪仅略高于随机。
- **Oasis**(Decart + Etched, 2024-10):diffusion transformer + diffusion-forcing 实时生成 Minecraft 式世界,H100 上 360p/20fps;**500M 参数权重开源**。
- **DIAMOND**(NeurIPS 2024 Spotlight):用 EDM diffusion 公式做世界模型,论点是"离散 latent 压缩会丢失对 RL 重要的视觉细节";Atari 100k 上 mean HNS **1.46**。

### 3.5 关键争议:视频模型真的懂物理吗?

两项独立研究——一项受控合成、一项真实世界基准——得出**一致**结论:

- **PhyWorld / How Far is Video Generation from World Model**(Kang et al., ByteDance Seed, ICML 2025, arXiv 2411.02385):自建 2D 简化几何模拟器(消除纹理),测经典力学(惯性、牛顿第二定律、弹性碰撞)。**核心结论:"视频生成即使扩大规模也未能从数据中学到物理规律。"** OOD 误差比 in-distribution 高一个数量级(如 0.427 vs 0.012),且扩数据/模型几乎无改善;机制是 **"case-based generalization"(模仿最近训练样本)而非抽象规则**;属性优先级 color > size > velocity > shape(源于像素空间变化幅度)。
- **Physics-IQ**(Motamed et al., INSAIT + Google DeepMind, arXiv 2501.09038, WACV 2026):396 段真实视频基准,覆盖流体/光学/固体力学/磁学/热力学。**核心结论:物理理解严重受限,且与视觉真实度无关——"visual realism ≠ physical understanding"**;顶尖模型 VideoPoet 仅得 24.1% Physics-IQ 分数。

**这是对"world simulator"叙事最有力的反驳**:纯观察式的视频生成,即便视觉逼真,也不等于掌握了可外推的物理与几何结构。

## 第四部分:核心问题——3D 理解如何服务于世界模型与具身智能

### 4.1 为什么 3D 几何对世界模型不可或缺:研究者的明确表述

Fei-Fei Li 在 *A Functional Taxonomy of World Models*(2026)中给出了迄今最系统的论述,提供大量可直接引用的原句:

- **语言 vs 世界的本质差异**:"The world is not made of words." 语言模型学的是文本的统计结构,世界模型学的是**空间与时间**的统计结构——光如何落在表面、物体如何响应力并遵循物理定律。
- **世界模型的三大功能分类**(基于 POMDP 循环 agent → action → state → observation):
  - **Renderer**:输出 observations / 像素,重视觉保真(如 Genie 3、World Labs RTFM)。致命短板——"carries no explicit understanding of three-dimensional structure … 建筑从上方看完美无缺,但试着开进城里,它们就散架了。"**纯像素渲染缺乏 3D 结构。**
  - **Simulator**:输出 **state**,即"几何、物理或动力学上忠实的表示";契约是**结构性的**:几何经得起检视、物理遵守牛顿定律;服务 RL agent / 机器人控制器 / 自动驾驶作为训练场。
  - **Planner**:输出 actions(VLA、model-based、World Action Models)。
- **为什么 simulation 是关键(linchpin)**:"If language is an abstraction of the world and pixels are a projection of it, then **geometry, physics, and dynamics are the world itself.**"
- **核心困难 = 3D 数据稀缺 + sim-to-real**:带显式几何、材质与物理标注的 3D 数据比互联网视频稀缺**数个数量级**;且生成式 simulator 新增风险——"AI 生成的几何看起来正确,却可能包含自相交或错误尺度,产生荒谬的物理。"

论文级佐证:**GAIA-1** 报告世界模型自发涌现"understanding of 3D geometry";**MILE** 显式"leverages 3D geometry as an inductive bias";**OccWorld** 选 3D occupancy 的理由(expressive / efficient / versatile);**3D-VLA** 批判 2D VLA"缺乏与 3D 物理世界的整合"。

### 4.2 自动驾驶:结构化 3D vs 像素级视频两条路线

自动驾驶是世界模型落地最深入的领域,清晰地分化为两条技术路线:

**结构化 3D 路线**:

- **MILE**(Wayve, NeurIPS 2022):联合学习世界模型与驾驶策略,纯离线;显式把 3D 几何作为 inductive bias,在"想象"中预测未来并解码为 BEV 语义分割;CARLA 新城镇+新天气下 driving score 较 SOTA 提升 31%。
- **OccWorld**(ECCV 2024):**基于 3D occupancy 的世界模型**。用 scene tokenizer 得离散 scene tokens → GPT-like spatial-temporal transformer 生成后续 scene + ego tokens → 解码未来 occupancy 与轨迹;在 nuScenes 上做 **4D occupancy forecasting** 与规划,**无需 instance/map 监督**。

**像素级视频生成路线**:

- **GAIA-1**(Wayve, 2023):9B 参数,4,700 小时驾驶数据,把 world modeling 转为离散 token 序列预测(video + text + action);报告涌现的 3D 几何理解。
- **GAIA-2**(2025-03):从逐帧自回归改为 **latent diffusion + video tokenizer**,保证多相机空间一致性;用于生成 safety-critical 场景压力测试。
- **GAIA-3**(2025-12)[厂商声称]:15B 参数,定位从"仿真"推进到"评估",新增 embodiment transfer;宣称 synthetic-test 拒绝率降 5×。
- **DriveDreamer / DriveDreamer-2**(ECCV 2024 / AAAI):首个从真实驾驶视频构建的世界模型,用 LLM 增强生成多样视频。

两条路线殊途同归:**从传感器观测预测未来场景/占据 → 在潜空间或占据空间"想象"多种 ego 轨迹后果 → 做 motion planning**。

### 4.3 具身智能:世界模型作为通用仿真器与数据引擎

- **UniSim — Learning Interactive Real-World Simulators**(Sherry Yang / Google DeepMind, ICLR 2024 Outstanding Paper):5.6B 参数,由 video diffusion 参数化;输入一张图 + 自然语言动作("open the drawer"),生成执行该动作的后续视频。作为 real-world-like 模拟器训练 high-level VLM planner 与 low-level RL policy,二者均展现显著 real-world transfer。**"世界模型作为通用交互式仿真器/数据引擎/policy learner"的核心范式论文。**
- **操纵世界模型近期趋势**(2024–2025,多为预印本,置信度中):用视频生成/流作为世界模型驱动操纵——Dreamitate(CoRL 2024)、3DFlowAction、NovaFlow、DiWA、World4RL 等。

**重要的现实校准**:Fei-Fei Li 本人坦承,近两年的机器人 demo"几乎全部局限于高度受限的实验室设置,物体集狭窄、任务时程短。没有一个在真实部署所要求的复杂度、多样性或持续时间上得到验证。"——这是区分炒作与实证的权威声音。

### 4.4 3D + 世界模型的直接融合:三种落点

- **生成落点 — 3D-VLA**(ICML 2024):把 3D 表示(点云)直接嵌入世界模型,构建于 3D-based LLM 之上,训练 embodied diffusion 预测**目标图像与点云**。
- **重建/数据落点 — DriveDreamer4D**(CVPR 2025):"世界模型是 4D 场景表示的高效数据引擎",用世界模型先验生成新轨迹视频来训练 **4D Gaussian Splatting**,解决传统传感器仿真在复杂机动下渲染失效。
- **综述地图 — 3D and 4D World Modeling: A Survey**(arXiv 2509.07996, 2025):首篇专门综述,给出 video-based / occupancy-based / LiDAR-based 的 taxonomy,聚焦用**原生 3D/4D 表示**(RGB-D、occupancy grids、LiDAR)做大规模世界建模。

### 4.5 收敛终局:统一世界模型

三条研究线正合流为一:**3D 生成(renderer)+ 空间智能/仿真(simulator)+ 世界模型/规划(planner)**。共识是它们所需的底层知识"largely the same——geometry, physics, dynamics",是同一底层理解的三种投影。

工程实证:

- **World Labs — Marble**(Fei-Fei Li, 2025-11)[部分为厂商/媒体]:首个商用生成世界模型,从 text/image/video/spatial sketch 生成可探索 3D 环境,**同时输出 Gaussian splats(供视觉探索)与 collision meshes(供物理引擎)**——Li 称其"溶解了 renderer 与 simulator 的边界"。
- **NVIDIA Cosmos**(2025-01, arXiv 2501.03575;Cosmos 3, 2025):World Foundation Model 平台,面向 Physical AI,作为 World Action Models 的 backbone。
- **DriveDreamer4D**:世界模型喂养 4DGS(renderer ↔ simulator 桥接)。

**终局假设**:一个统一的基座世界模型,能按下游需求切换 render / simulate / plan 三种输出。这尚属方向性主张,统一模型远未实现。

## 第五部分:整体判断与研究启示

1. **表示层的收敛**:3D 重建的"前馈化"(VGGT、LRM)与 3D 生成的"原生化"(CLAY、TRELLIS)正在提供可实时、可泛化、结构化的 3D 场景表示,这恰是世界模型所缺的"可检视状态"。

2. **世界模型的路线之争尚未收敛**:生成式像素预测(Sora / Genie / GameNGen)在视觉与可交互性上领先,但被 PhyWorld / Physics-IQ 证明**未真正学到物理**;表征式潜空间预测(JEPA)与结构化 3D(occupancy / 点云 / 4DGS)在物理一致性与规划效率上更有前景,但生成能力与生态成熟度落后。二者可能互补而非互斥。

3. **3D 之于具身智能的价值是结构性的,而非锦上添花**:几何一致性、可物理仿真的状态、sim-to-real,是把"看起来对"变成"用起来对"的必要条件。3D 数据稀缺与 sim-to-real gap 是当前最硬的瓶颈——这既是挑战,也是 3D 生成/重建研究能贡献合成训练数据的最大机会点。

4. **给研究者的可操作方向**:(a) 面向世界模型的**几何一致性约束/评测**(防止自相交、错误尺度);(b) 用大规模 3D 生成模型**合成带物理标注的训练数据**缓解数据稀缺;(c) 探索 **JEPA 式表征预测 + 显式 3D 结构**的混合世界模型;(d) 建立超越视觉真实度的**物理/几何基准**(Physics-IQ、PhyWorld、VideoPhy-2 的方向)。

## 置信度与局限说明

- **高置信度**:所有核心论文的方法机制、作者、发表场所、里程碑(NeRF / 3DGS / DreamFusion 系 / Dreamer 系 / JEPA / GameNGen / DIAMOND / PhyWorld / Physics-IQ / UniSim / OccWorld / MILE / 3D-VLA 等)均有一手论文或权威来源;DreamerV3 的 Nature 2025 发表与 V-JEPA 2 的 arXiv 2506.09985 已单独核实。
- **中置信度**:2025–2026 的商业/厂商动态(GAIA-3 拒绝率降 5×、Cosmos 3 采用方、Genie 3 长时一致性、World Labs 融资数字、Rodin/Hunyuan 商业性能)多来自厂商公告与科技媒体,未见独立第三方复核,报告中已标注"[厂商声称]"。
- **低-中置信度**:2025–2026 的机器人操纵世界模型预印本(3DFlowAction、NovaFlow、DiWA、World4RL 等)方向有代表性但结果未复核。
- **已剔除**:搜索返回中出现的若干未来日期 arXiv 编号(2604.x / 2605.x / 2606.x 等)疑为占位/异常,未作为核心证据引用。
- 各方法自报的"最快速度"(5s / 0.5s / 0.1s / 100 FPS)基于不同 GPU 与设置,不可横向硬比。
